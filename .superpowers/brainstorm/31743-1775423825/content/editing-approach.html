How should inline editing work on the board?

A dispatcher needs to fix a wrong type, priority, or location on an existing incident.

A

Side Drawer (slide-in panel)

Click anywhere on a row → a panel slides in from the right showing all fields as editable inputs. The board stays visible behind it. Save/Cancel at the bottom of the drawer.

Pros

  • Plenty of room for all fields
  • Board context stays visible
  • Natural place for full description + audit trail

Cons

  • Extra click to open
  • Covers part of board
B

Edit Modal (dialog popup)

A dedicated "Edit" button per row opens a centered modal with all editable fields. Board is blocked until modal is dismissed. Status workflow stays on the row.

Pros

  • Clear focus — nothing else clickable
  • Standard, familiar UX

Cons

  • Board disappears behind overlay
  • Extra button clutters each row
C

Expandable Row

Click a row → it expands in-place to show editable fields below the summary line. The rest of the board shifts down. No overlay, no navigation away.

Pros

  • No overlay — full board visible
  • Spatially connected to the row
  • Fast, no navigation

Cons

  • Limited vertical space per field
  • Board layout shifts when expanded
