What's missing — where should we focus?

Select the areas you want to tackle. I'll design a plan around your priorities.

A

Inline Editing on the Board

After saving an incident, dispatchers can only advance its status. They can't fix a wrong type, priority, location, or people-involved without deleting and re-creating it. The board needs in-place editing for those fields.

B

Text Search + Better Filtering

The board only has dropdown filters (status / type / priority). There's no way to search descriptions, filter by date range, or sort columns. Hard to find a specific incident in a busy shift.

C

2D Hour × Day Heatmap

The backend already returns hour + day-of-week data. The UI only shows a 1D bar chart (total by hour). A full grid would reveal patterns like "Friday nights at 11pm are dangerous" that the bar chart hides.

D

Incident Detail View / Drill-Down

Clicking a row on the board does nothing. There's no way to see the full raw description, AI rationale, pattern flag detail, audit trail, or full history of changes for a single incident.

E

Backend Robustness

AI calls (classify + digest) silently swallow rate limit errors, auth failures, and timeouts — they log an empty string and the frontend shows a generic error. Pagination is also missing: all incidents load at once.

F

UX Polish

Small but impactful: timestamps only show HH:MM (yesterday's incidents look current), the stale-P1 banner never auto-dismisses, export has no feedback, and the pattern flag uses an emoji instead of a proper icon.
